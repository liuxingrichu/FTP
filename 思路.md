
用户信息存放在batabase目录，init_db.py文件用于生成用户账号和密码、磁盘配额和磁盘使用信息

FTP客户端

	功能：
		authenticate	认证
		push			文件上传
		pull			文件下载
		ls				查看目录下的内容
		cd ..			进入前一层目录
		cd xx			进入xx目录
		pwd				查看当前路径
		interaction		先进行用户认证，再允许用户进行相应操作
		help			用户输入提示
		quit			用户退出
		show_bar		传输进度条

FTP服务器

	功能：
		authenticate	处理用户认证
		push			配合客户端文件上传
		pull			配合客户端文件下载	
		cd				进行用户路径切换，权限判定
		ls				配合客户端显示当前目录下内容
		quit			用于记录用户退出信息
		handle			业务处理流程
==============================================
客户端文件上传流程
	（1）合法性判定
	（2）组合信息，发送数据
	（3）等待服务器反馈
	（4）根据反馈数据，判定用户磁盘空间
	（5）进行文件传输和MD5
	（6）将文件MD5发送给服务器
	（7）通过服务器反馈数据，判定是否传输成功
----------------------------------------------
服务端文件上传流程
	（1）用户磁盘空间判定
	（2）文件接收目录设置
	（3）同名文件判定
	（4）文件接收及MD5
	（5）文件一致性判定
	（6）若文件接收成功，反馈信息给客户端及更新用户磁盘使用信息
	（7）若文件接收失败，反馈信息给客户端及删除接收失败文件
